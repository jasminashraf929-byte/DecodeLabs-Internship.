Pillar 1 — The Blueprint

Database Schema Design

Two tables power the system. Each column is typed and constrained directly at the schema level — the database is the final source of truth.

💬 messages
id TEXT PRIMARY KEY
first_name TEXT CHECK ≥ 2
last_name TEXT NOT NULL
email TEXT NOT NULL
subject TEXT DEFAULT 'General'
message TEXT CHECK ≥ 10
submitted_at TEXT NOT NULL
📧 subscribers
id TEXT PRIMARY KEY
email TEXT UNIQUE
subscribed_at TEXT NOT NULL

CRUD → HTTP Mapping

CREATE→POST /api/contact
READ→GET /api/messages
UPDATE→PUT /api/messages/:id
DELETE→DEL /api/messages/:id
